# Phase 02.1 Plan 04: Station Integration Summary

Wired `vision_confirm.find_robot_qr` + `compute_drift` into `station._run_cycle` at two post-motion points, gated by `config.vision_confirm_enabled`, recording all three new optional JSONL fields via the already-extended `EventLogger.write`.

## What Was Built

Two 4-line vision-confirm hooks inserted into `station._run_cycle`:

1. After `robot.move_to_bin(dest_bin)` — calls `_vision_check(frame, f"bin_{dest_bin}")` → result stored in `vc_bin`
2. After `robot.return_home()` — calls `_vision_check(frame, "home")` → result stored in `vc_home`

The home-check result (`vc_home`) is written to `EventLogger.write(vision_confirmed=, drift_px=, vision_reason=)`. When `vision_confirm_enabled=False` both tuples are `(None, None, None)` and `LogEntry.to_dict()` omits the three fields — Phase 1 JSONL is byte-identical.

### `_vision_check(self, frame, target_name)` helper

All 6 reason codes are handled inside `_vision_check`:

| Reason | Condition |
|---|---|
| `ok` | drift <= tolerance |
| `drift_exceeded` | drift > tolerance (D-14, cycle continues) |
| `robot_qr_not_found` | `find_robot_qr` returns None or frame is None (D-09) |
| `no_target_calibrated` | target entry is None in config (D-17) |
| `error:<ExcType>` | any exception from pyzbar (D-15, observability only) |
| `None` | feature disabled — fields omitted from JSONL entirely |

### D-17 startup warning

`Station.__init__` checks `vision_confirm_enabled=True` with all-null targets and logs exactly one warning: `"vision_confirm_enabled=true but no targets calibrated — skipping checks"`.

### Lock invariant preserved

The `try: ... finally: self._cycle_lock = False` block is unchanged. `_vision_check` wraps `find_robot_qr` in `try/except Exception` so pyzbar crashes cannot escape to the finally block — the lock is always released.
