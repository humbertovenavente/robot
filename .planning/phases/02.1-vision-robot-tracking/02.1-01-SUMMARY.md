# Phase 02.1 Plan 01: Config + EventLog + Requirements Summary

**One-liner:** Added vision_confirm_enabled/tolerance_px/robot_vision_targets to Config, three optional vision kwargs to EventLogger.write with None-omission in JSONL, and verified VIS-05..08 in REQUIREMENTS.md — all purely additive, zero Phase 1/2 behavior change.

## What Was Built

### Task 1: Config fields + YAML keys

Three new fields added to `config.py` (Pydantic model):

| Field | Type | Default | Decision ref |
|-------|------|---------|-------------|
| `vision_confirm_enabled` | `bool` | `False` | D-16 |
| `vision_confirm_tolerance_px` | `int` | `30` | D-13 |
| `robot_vision_targets` | `Dict[str, Optional[List[int]]]` | all-null dict | D-11 |

`robot_vision_targets` has a `field_validator` that rejects non-2-element coordinate lists at load time (fail-fast at startup, not at runtime).

`station_config.yaml` extended with four new keys appended at the end:

```yaml
vision_confirm_enabled: false
vision_confirm_tolerance_px: 30
robot_vision_targets:
  home: null
  bin_1: null
  bin_2: null
  bin_3: null
```

With these defaults, loading the file produces a Config identical in behavior to Phase 1 — the vision code path is never entered when `vision_confirm_enabled=false`.

### Task 2: LogEntry + EventLogger.write vision fields

`LogEntry` dataclass extended with three optional fields (all default `None`):
- `vision_confirmed: Optional[bool]`
- `drift_px: Optional[int]`
- `vision_reason: Optional[str]`

`to_dict()` implements **None-omission**: when all three are `None`, they are stripped from the output dict, so the JSONL wire format is byte-identical to Phase 1 D-16 schema. Phase 1 log consumers that validate strict key-sets do not break.

`EventLogger.write` extended with a keyword-only boundary (`*`) followed by the three optional kwargs. All existing positional callers (`write("A", 1, 4500, "completed", None)`) continue to work unchanged.
