# Phase 02.1 Plan 03: Calibrate Pixel Capture Summary

Pixel-capture sub-flow integrated into `calibrate.py` via TDD: operator sets encoder targets (Phase 1 ROB-05) and pixel targets (VIS-06) in a single interactive calibration run, with retry/skip UX and yaml round-trip that preserves all existing keys.

## What Was Built

### capture_pixel_target(cap, target_name, max_retries=3)

Grabs one frame from the open `cv2.VideoCapture`, runs `find_robot_qr` from `vision_confirm.py`. On detection, prints the confirmed center and returns it. On miss, prompts the operator `[r]etry / [s]kip` up to `max_retries` times; skip returns `None` (D-10: null-on-skip is valid, runtime will no-op that target per D-17).

### write_pixel_target_to_yaml(yaml_path, target_name, center)

`yaml.safe_load` → mutate `robot_vision_targets[target_name]` → `yaml.safe_dump(sort_keys=False)`. Uses `dict.setdefault` to create the `robot_vision_targets` subtree if absent. All other top-level keys (`class_to_bin`, `camera_index`, `orchestrator_url`, `home_encoder_target`, etc.) pass through untouched.

### run_calibration integration

`run_calibration` gains an optional `cap` parameter (default `None`). When `None` (CI, `--yes`, `--stub-preset`, `--dry-run`) the pixel-capture path is entirely skipped — Phase 1 ROB-05 behavior is bit-for-bit identical. When a `cv2.VideoCapture` is provided and `dry_run=False`, the sub-flow fires at each step (home + bin_1 + bin_2 + bin_3) immediately after the encoder reading, writes the result to yaml, and prints a summary:

```
Pixel targets: 3/4 captured — bin_3 skipped
```

## UX Flow (operator interactive mode)

```
1. Move the robot to HOME position. Press Enter when ready...
  HOME encoder = 0
  pixel target for home: (320, 240)

2. Move the robot to BIN 1. Press Enter when ready...
  BIN 1 encoder = 90
  ROBOT QR not found at bin_1. [r]etry / [s]kip: r
  pixel target for bin_1: (420, 250)

...

Wrote calibration to station_config.yaml
Reminder: restart station.py to pick up the new values (D-14).
Pixel targets: 3/4 captured — bin_3 skipped
```

## yaml Write Strategy

- `safe_load` → plain `dict` (preserves insertion order in Python 3.7+)
- `setdefault("robot_vision_targets", {...})` creates the subtree only if missing
- `safe_dump(sort_keys=False, default_flow_style=None)` minimises diff churn
- Only `robot_vision_targets.<key>` changes; all sibling keys are byte-identical after round-trip

## Demo-Day Note

The ROBOT QR printed on top of the robot should be **2x larger** than the package QRs (A/B/C) for reliable full-frame decode at venue lighting. Document this on the demo-day setup checklist.
